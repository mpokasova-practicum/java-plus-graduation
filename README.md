## О проекте
**ExploreWithMe** - это приложение, который позволяет пользователям делиться информацией об интересных событиях и находить компанию для участия в них.
Приложение представляет собой афишу. В этой афише можно предложить какое-либо событие от выставки до похода в кино и собрать компанию для участия в нём.

## Архитектура

### Компоненты системы

1. **Discovery Server (Eureka)**:
    - Реализует сервис-ориентированное обнаружение.
    - Позволяет микросервисам регистрироваться и находить друг друга.

2. **Config Server**:
    - Централизованное хранилище конфигураций.
    - Упрощает управление настройками всех микросервисов.

3. **Gateway Server**:
    - API Gateway для маршрутизации запросов.
    - Инкапсулирует внутреннюю архитектуру, предоставляя единый входной API.

4. **Stats Service**:
    - Собирает и анализирует статистику по запросам пользователей.

5. **Event Service**:
    - Управляет событиями, категориями,подборками и локацией!

6. **Request Service**:
    - Управляет заявками на участие в мероприятиях.

7. **User Service**:
    - Сервис для администратора - управления пользователями.

8. **Comment Service**:
    - Управляет комментариями пользователей.

---

## Взаимодействие между сервисами

Микросервисы взаимодействуют друг с другом через:
- **OpenFeign** – для межсервисной коммуникации.
- **Eureka** – для динамического поиска сервисов.
- **API Gateway** – для маршрутизации внешних запросов к соответствующим сервисам.

Каждый микросервис обладает собственной базой данных на основе PostgreSQL и взаимодействует с другими сервисами через REST API.

## Фича
В качестве фичи были выбраны комментарии. Максиимальная длина составляет 1000 знаков. Создавать/редактировать/удалять комментарий может авторизованный пользователь, не являющийся автором мероприятия. Администратор модерирует комментарии перед публикацией. Информация о самом проекте и его эдпоинтах представлена ниже.

### **Эндпоинты фичи**

| Метод  | Путь                                 | Описание                    |
| ------ | :----------------------------------- | --------------------------- |
| POST   | /users/{userId}/comments             | Создать пользовательский комментарий         |
| DELETE | /users/{userId}/comments/{commentId} | Удалить пользовательский комментарий         |
| PATCH  | /users/{userId}/comments/{commentId} | Отредактировать пользовательский комментарий |
| DELETE | /admin/comments/{commentId}          | Удалить комментарий пользователя        |
| PATCH  | /admin/comments/{commentId}          | Отредактировать комментарий пользователя |
| GET    | /admin/comments                      | Получить все комментарии        |

## Конфигурация

Все конфигурации для микросервисов хранятся централизованно в **Config Server**. Взаимодействие настроено через Spring Cloud Config, который обеспечивает динамическое получение конфигурации.
